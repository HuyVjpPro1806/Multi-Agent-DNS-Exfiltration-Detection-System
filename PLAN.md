## 1. Project Overview

Build a multi-agent pipeline that analyses DNS traffic, detects
covert channels and data exfiltration, and generates a structured
threat report. The pipeline covers 3 sequential stages with parallel
execution at Stage 2.

## 4. Stage Detail

### Stage 1 — Data Ingestion

**Goal:** Accept PCAP or CSV input, normalize to `dns_queries.json`.

**Data flow:**
- PCAP path → `pcap_reader_agent` → `raw_packets.json`
  → `dns_extractor_agent` → `dns_queries.json`
- CSV path → `dns_extractor_agent` (direct) → `dns_queries.json`

**Dataset:**
- CSV: `daumel/dns-tunneling-dataset` on Kaggle
  (BCCC-CIC-Bell-DNS-2024, 9 tunneling tools, benign + malicious labels)
- PCAP: generated locally by `generate_test_pcap.py` using scapy

**Output schema (`dns_queries.json`):**
```json
{
  "query_id": 1,
  "timestamp": 0.0,
  "src_ip": "10.0.0.5",
  "domain": "a3f9bc12.evil.com",
  "query_type": "A",
  "subdomain": "a3f9bc12",
  "tld": "com",
  "label_count": 3,
  "domain_length": 18,
  "digit_ratio": 0.625,
  "label": "malicious",
  "count": 1,
  "source": "pcap"
}
```

### Stage 2 — Parallel Analysis

**Goal:** Score each domain with 3 independent methods simultaneously.

**Parallelism:** All 3 agents declared under `/parallel` in `dns_exfil.chain.md`.
They read the same `dns_queries.json` with no data dependency between them.

#### Agent 2a — `entropy_agent`
- Tool: `shannon_entropy.py`
- Method: Shannon entropy H = -Σ p(x) log₂ p(x) on subdomain characters
- Threshold: entropy > 3.5 → suspicious
- Output field: `entropy_score` (float)
- No training required

#### Agent 2b — `dga_classifier_agent`
- Tool: `dga_model.py`
- Method: RandomForest classifier trained on CSV Kaggle dataset
- Features: `domain_length`, `digit_ratio`, `label_count`, `subdomain` length,
  vowel ratio, consonant ratio, unique char ratio
- Output field: `dga_score` (float 0.0–1.0, probability of malicious)
- Requires: `models/dga_model.pkl` (train offline before demo)

#### Agent 2c — `embedding_agent`
- Tool: `embed_score.py`
- Method: TF-IDF character n-gram vectorizer + cosine distance from benign centroid
- Train: fit vectorizer on benign domains from CSV, save centroid vector
- Output field: `embed_score` (float 0.0–1.0, distance from benign cluster)
- Requires: `models/embed_model.pkl` (train offline before demo)

**Output schema (`scores.json`):**
```json
{
  "query_id": 1,
  "domain": "a3f9bc12.evil.com",
  "label": "malicious",
  "entropy_score": 4.6,
  "dga_score": 0.91,
  "embed_score": 0.87
}
```

### Stage 3 — Orchestration + Report

**Goal:** Aggregate 3 scores, classify each domain, generate threat report.

#### Agent 3a — `orchestrator_agent`
- Tool: `aggregate_scores.py`
- Method: weighted average of 3 scores
  - weight: entropy 0.3, dga 0.4, embed 0.3
- Threshold: combined_score > 0.6 → suspected exfiltration
- Output: enriched `scores.json` with `combined_score` and `verdict` fields

#### Agent 3b — `report_agent`
- Tool: `generate_report.py`
- Output: `data/output/exfil_report.md`
- Report sections:
  1. Summary (total queries, suspected count, date)
  2. Top 10 suspicious domains table (sorted by combined_score)
  3. Score breakdown per domain (entropy / dga / embed)
  4. Source breakdown (pcap vs csv)
  5. Recommendations
